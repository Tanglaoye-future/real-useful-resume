# ResuMiner vNext PRD（外企优先）

## 1. 背景
- 大厂岗位已基本投递完成，当前核心目标转为“上海外企高价值实习机会”。
- 现有第三方链路（51job/猎聘）历史覆盖不足，且运行依赖（RPC/Redis/登录态）不稳定。

## 2. 目标
- 在 24 小时内交付可投递的“严格外资公司”岗位清单。
- 明确剔除八大厂岗位：字节、腾讯、快手、小红书、美团、阿里、京东、哔哩。
- 清理项目噪音文件，确保仓库可维护、可复现。

## 3. 高价值定义（当前版本）
- 公司优先：先判断是否为严格外资/跨国企业主体。
- 岗位次优：数据分析/商业分析/BI/策略相关优先。
- JD加分：Python、SQL、统计、实验、可视化、策略分析等关键词。
- 负向剔除：外包/中介/纯销售类岗位。

## 4. 范围
- 数据源：51job、猎聘（第三方）。
- 城市：上海。
- 岗位类型：实习/校招（intern）。

## 5. 关键链路
1. 启动 Redis（6379）
2. 启动 RPC 服务（5600）
3. 校验登录态（猎聘）
4. 运行第三方抓取
5. 严格外资过滤与价值排序
6. 链接校验与可投递清单输出

## 6. 交付产物
- `release_data/foreign_strict_shanghai_opportunities_latest.csv`
- （后续）`release_data/foreign_strict_shanghai_verified_only.csv`
- （后续）`release_data/foreign_strict_top50_actionable.csv`

## 7. 成功标准
- 第三方抓取链路稳定（可持续产出）。
- 严格外资岗位清单可复现。
- 仓库无缓存噪音文件入库（`__pycache__`、`.pyc`、临时检查点）。

## 8. 风险与缓解
- 反爬/风控：采用 RPC + 会话复用 + 节流 + 重试。
- 登录态失效：要求登录后立即进行 audit 抓取验证。
- 数据覆盖波动：多轮关键词与分页策略迭代。
