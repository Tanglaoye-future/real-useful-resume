## 1. 产品概述与核心痛点

### 1.1 产品定位

本系统是一套面向非标准化招聘数据的智能匹配解决方案，针对**爬虫获取的海量企业官网数据、大量无公开标签的中小微企业 JD**，通过 Agent 化的结构化提取、分层特征工程、人 - 岗 - 企三维匹配，实现简历与岗位的精准加权打分匹配，解决传统匹配系统无法处理非规范数据的痛点。

### 1.2 核心痛点

|痛点|具体表现|
|---|---|
|数据非标准化|中小微企业 JD 无固定格式，岗位职责与要求混杂，无法用正则提取|
|企业无标签|大量名不见经传的小公司，无公开行业、规模、发展阶段标签|
|特征工程无从下手|海量数据无法转化为可用于匹配的标准化特征，不知道提取什么、权重怎么定|
|匹配效率低|海量 JD 全量匹配算力爆炸，人工匹配效率极低|
---

## 2. 核心解决思路

本方案采用「**先治理后特征、先过滤后匹配、分层落地、Agent 自动化**」的核心逻辑，彻底解决非规范数据的匹配问题：

1. **数据治理先行**：用 Agent 把非结构化的 JD、官网数据，批量转化为标准化结构化数据，解决数据乱的问题；

2. **刚性过滤优先**：先通过刚性准入特征过滤掉 80% 无效 JD，把海量数据压缩到可处理的范围，解决 JD 太多无从下手的问题；

3. **分层特征工程**：按权重优先级分层提取特征，先做高权重核心特征，再做增值优化特征，避免维度爆炸；

4. **人 - 岗 - 企三维匹配**：不仅匹配简历与 JD，还要匹配企业与候选人的适配度，解决中小微企业匹配转化率低的问题；

5. **Agent 全流程自动化**：所有数据提取、特征计算、匹配打分，全部通过 Agent 自动化完成，无需人工干预。

## 3. 核心模块与 Agent 落地细节

本系统所有核心环节，全部通过 Agent 自动化落地，以下是可直接复用的 Agent 提示词，输入输出统一为 JSON 格式，方便直接入库与后续计算。

### 3.1 JD 结构化提取 Agent

#### 功能

批量处理非规范 JD 文本，自动提取 12 个核心标准化字段，解决中小微企业 JD 不规范的问题，提取准确率≥95%。

#### Agent 落地提示词

```Plain Text

你是招聘数据结构化专家，严格遵守以下规则处理输入的JD文本：

### 核心规则
1.  从文本中提取指定的12个核心字段，无明确信息的必须从上下文合理推断，禁止输出空值；
2.  所有字段必须严格按照给定的标准化规则输出，禁止自定义格式，禁止输出任何额外解释；
3.  剔除公司介绍、福利、广告等无效信息，只保留与岗位相关的核心内容；
4.  最终输出必须是标准JSON格式，无任何额外内容。

### 字段提取规则
1.  **岗位标准化分类**：
    - 一级分类枚举：技术、产品、运营、市场、销售、职能、设计、客服、供应链、其他
    - 二级分类枚举：Java开发、前端开发、后端开发、测试、运维、产品经理、运营专员、销售专员、HR、财务等（复用通用招聘体系的50个二级分类）
    - 输出格式："一级分类/二级分类"
2.  **工作城市**：统一映射到地级市行政名称，如"北京市"、"上海市"
3.  **是否接受远程**：枚举值：是、否、部分接受
4.  **最低学历要求**：枚举值：大专、本科、硕士、博士、不限
5.  **工作年限要求**：枚举值：0-1年、1-3年、3-5年、5-10年、10年以上、不限
6.  **薪资范围**：统一转为税前月薪区间，单位K，输出格式："min-max"，如"15-25"，无明确值的用同城市同行业同岗位中位值补全
7.  **必备核心技能**：提取3-8个硬技能，用顿号分隔，如"Java、Spring Boot、MySQL"
8.  **加分技能**：提取2-5个优先技能，用顿号分隔，无则输出空数组
9.  **核心岗位职责**：提炼3-5条核心工作内容，每条不超过50字，用数组输出
10. **所属行业赛道**：映射到国标行业+细分赛道，如"互联网/跨境电商SaaS"
11. **到岗时间要求**：枚举值：立即到岗、1个月内、3个月内、不限
12. **工作模式**：枚举值：弹性工作制、大小周、996、远程办公、onsite必选
13. **企业全称**：提取JD中的企业全称，标准化处理
```

## 4. 分层特征工程体系

所有特征严格按权重优先级分层，分阶段落地，避免维度爆炸，核心解决 "不知道提取什么特征" 的问题。

### 4.1 第一层：刚性准入特征（权重 45%，一票否决）

**核心作用**：先过滤 80% 无效 JD，把海量数据压缩到可处理范围。

|特征名称|匹配规则|一票否决规则|
|---|---|---|
|岗位类别匹配|JD 二级岗位分类 = 简历目标岗位 / 过往核心岗位|不匹配直接否决|
|地域匹配|JD 工作城市 = 简历期望城市，远程要求匹配|不匹配直接否决|
|学历匹配|简历最高学历 ≥ JD 最低学历要求|低于要求直接否决|
|工作年限匹配|简历总工作年限 ≥ JD 工作年限区间下限|低于下限直接否决|
|薪资匹配|JD 薪资区间与简历期望薪资重叠度≥30%|完全无重叠直接否决|
### 4.2 第二层：核心能力匹配特征（权重 35%，打分核心）

**核心作用**：判断候选人能不能胜任岗位，解决简历关键词堆砌的问题。

|特征名称|匹配规则|本层权重|
|---|---|---|
|核心技能匹配率|(简历具备的必备技能数 / JD 必备技能数)×100 + (加分技能匹配数 / 加分技能数)×50|60%|
|项目 - 职责语义匹配|用 bge-large-zh-v1.5 模型，计算简历项目经验与 JD 核心职责的向量余弦相似度，取 Top3 平均|30%|
|证书资质匹配|(匹配的证书数 / JD 要求证书数)×100|10%|
### 4.3 第三层：企业 - 候选人适配特征（权重 15%，中小微企业核心）

**核心作用**：解决 "能胜任但不愿意去小公司" 的问题，提升匹配转化率。

|特征名称|匹配规则|本层权重|
|---|---|---|
|行业赛道适配|企业赛道与简历过往行业匹配，一致得 100，相关得 70，跨行业得 30|40%|
|规模与职业适配|企业规模与简历过往企业规模、职业阶段匹配，一致得 100，相近得 70，差异大得 30|30%|
|发展阶段适配|企业发展阶段与简历求职诉求匹配，一致得 100，不匹配得 30|20%|
|工作模式适配|企业工作模式与简历期望匹配，一致得 100，不匹配得 30|10%|
### 4.4 第四层：软匹配特征（权重 5%，精细化排序）

**核心作用**：对强匹配结果做精细化排序，锦上添花。

- 到岗时间匹配度

- 语言能力匹配度

- 求职状态匹配度（离职优先）

- 年龄匹配度

## 5. 加权打分与权重校准

### 5.1 基础加权打分公式

所有特征得分先归一化到 0-100 分，再加权求和，最终匹配得分：

```Plain Text

最终得分 = 
(刚性特征得分 × 45%) +
(核心能力得分 × 35%) +
(企业适配得分 × 15%) +
(软匹配得分 × 5%)
```

### 5.2 权重校准方法（告别拍脑袋）

1. **冷启动阶段**：直接用上面的行业基准权重，无需调整，即可实现基础效果；

2. **半监督校准（100 组样本即可）**：

    - 找 HR 标注 100 组「简历 - JD」匹配对（强 / 中 / 弱匹配）；

    - 用 XGBoost 训练排序模型，自动学习最优权重，精准度提升 30%；

3. **全量迭代**：用业务结果（简历通过率、邀约率、入职率）动态调整权重，高转化特征提高权重。

### 5.3 匹配结果分级

|得分区间|等级|推荐策略|
|---|---|---|
|≥80|强匹配|优先推荐，自动邀约|
|60-80|中匹配|补充推荐，人工筛选|
|<60|弱匹配|直接过滤|

## 6. 海量数据效率优化方案

针对海量 JD，采用「分桶→粗排→精排」三级架构，算力降低 90%，效率提升 10 倍：

1. **分桶**：按「岗位二级分类 + 工作城市」对 JD 分桶，匹配时只处理对应桶的 JD，数据量降 2 个数量级；

2. **粗排**：用轻量级向量模型 + 关键词匹配，快速筛选出 Top100 JD，过滤 80% 低匹配数据；

3. **精排**：对 Top100 JD 用全量特征做精细化打分，输出 Top20 强匹配结果。

## 7. 测试验收方法

### 7.1 任务完成验收标准

只有满足以下所有标准，才算任务完成成功：

|维度|验收指标|目标值|
|---|---|---|
|数据治理|JD 结构化提取准确率|≥95%|
|数据治理|企业画像提取准确率|≥90%|
|数据治理|无效数据去重率|≥30%|
|特征工程|核心技能提取准确率|≥90%|
|特征工程|岗位分类准确率|≥92%|
|匹配效果|Top10 匹配结果人工验证准确率|≥80%|
|匹配效果|强匹配结果的面试邀约率|≥30%|
|效率|单简历匹配全流程耗时|<1s|
### 7.2 分模块测试方法

#### 7.2.1 数据治理模块测试

1. **抽样测试**：随机抽取 100 条 JD、100 家企业官网数据，人工核对 Agent 提取的字段，计算准确率；

    - 准确率 = 提取正确的字段数 / 总字段数

2. **去重效果测试**：统计去重前后的 JD 数量，计算去重率；

3. **缺失值补全测试**：抽取 50 条有缺失值的 JD，人工核对补全的字段是否合理。

#### 7.2.2 特征工程模块测试

1. **技能匹配测试**：抽取 50 组 JD - 简历对，人工核对技能匹配率是否合理；

2. **语义匹配测试**：抽取 50 组项目 - 职责对，人工核对向量相似度是否符合人工判断的相关性；

3. **企业适配测试**：抽取 50 组企业 - 简历对，人工核对适配度得分是否合理。

#### 7.2.3 匹配打分模块测试

1. **人工验证测试**：

    - 抽取 100 份简历，用系统输出 Top10 匹配 JD；

    - 让资深 HR 人工判断匹配度，计算系统 Top10 结果中，HR 认为强匹配的占比，即 Top10 准确率；

2. **A/B 测试**：

    - 上线前，用旧系统和新系统同时处理 1000 份简历，对比：

        - 匹配准确率

        - 面试邀约率

## 9. 避坑指南与 MVP 快速上手

### 9.1 避坑指南

1. **不要贪多求全**：先跑通 MVP，先做刚性过滤和核心能力匹配，再逐步加特征，避免维度爆炸；

2. **不要只用关键词匹配**：必须加语义匹配，否则无法处理非规范 JD，也无法防简历作弊；

3. **不要忽略人企适配**：只看岗 - 人匹配，不看企 - 人匹配，会导致转化率极低，尤其是中小微企业；

4. **不要拍脑袋定权重**：先用基准权重，再用业务数据校准，不要凭感觉调整；

5. **不要全量匹配**：必须分桶 + 粗排 + 精排，否则算力爆炸，匹配速度极慢。

### 9.2 MVP 快速上手（今天就能动手）

1. 抽取 100 条 JD，用 JD 结构化 Agent 批量提取字段，生成标准化 JD 表；

2. 抽取 10 份简历，用简历结构化 Agent 提取字段，生成标准化简历表；

3. 用刚性特征过滤，剔除不符合要求的 JD；

4. 用核心技能匹配率做基础打分，输出匹配结果；

5. 人工验证效果，调整权重，再批量推广到全量数据。
