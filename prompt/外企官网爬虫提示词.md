# 外企官网爬虫 — 实现提示词

> 本提示词用于驱动 LLM 在本仓库内实现一个独立的外企官网爬虫（与现有的 `official_multi_crawler.py` 并行存在，不互相影响）。

---

## 1. 目标

从 `rules/foreign_official_sites_seeds.json` 列出的 **28 家外企官网**抓取 **上海** 的 **校招/实习** 岗位，输出格式与现有 `outputs/raw/<company>_official_raw.csv` **完全一致**，以便下游可统一接入 `official_multi_crawler.py` 的快照/diff/聚合流程。

---

## 2. 范围

| 维度 | 取值 |
| --- | --- |
| 城市 | `上海` / `Shanghai`（详情页 `city` 字段或 `jd_raw` 命中即可） |
| 岗位类型 | 校招、实习、应届、`intern`、`graduate`、`新人`、`management trainee`（任一命中即可） |
| 数据源 | `rules/foreign_official_sites_seeds.json`（28 家，分 internet / fmcg / hedge_fund 三类） |
| 优先级 | P0（必须跑通且数据质量达标）→ P1（先通用爬，后续按需上专用） → P2（通用即可） |

---

## 3. 输入

1. **种子文件**：`rules/foreign_official_sites_seeds.json`
   - schema：`schema_version, scope, defaults, seeds[]`
   - 每条 seed：`company, industry, priority, ats_type, start_urls, allow_domains, notes`
2. **可参考的现有代码**（**只读不改**）：
   - `official_multi_crawler.py` — 中国互联网大厂的爬法、CSV schema、快照逻辑
   - `parsers/company_registry.py` / `parsers/company_adapters.py` — 适配器注册模式
   - `outputs/raw/alibaba_official_raw.csv` 等 — 字段样例

---

## 4. 输出

### 4.1 文件位置

| 文件 | 说明 |
| --- | --- |
| `outputs/raw/foreign_<slug>_official_raw.csv` | 每家公司一份原始 CSV，`<slug>` 形如 `google` / `pg` / `citadel` |
| `outputs/raw/foreign_official_aggregated.csv` | 全部 28 家合并后的总表（同 schema） |
| `logs/foreign_official_<TS>.log` | 心跳日志，每家公司一行进度 |

### 4.2 CSV schema（**严格沿用 `outputs/raw/alibaba_official_raw.csv` 的 20 列，顺序一致**）

```
url, company, name, city, jd_raw, salary, company_size, duration, academic,
publish_time, deadline, collect_time, source, recruit_type, raw_tags,
external_job_id, update_time, publish_time_source, deadline_source, sync_status
```

字段填充规则：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `url` | ✅ | 详情页绝对 URL |
| `company` | ✅ | 取自 seed 的 `company` 字段 |
| `name` | ✅ | 岗位标题 |
| `city` | ✅ | 标准化为 `上海`（即便页面写 `Shanghai` 或 `Shanghai, China`） |
| `jd_raw` | ✅ | 详情页主体文本（去重空白、保留换行）；至少 50 字 |
| `salary` |  | 没有就留空，不要瞎猜 |
| `publish_time` |  | ISO `YYYY-MM-DD HH:MM:SS`；缺失填 `不知道发布时间` |
| `deadline` |  | 同上规范；缺失填 `未知截止时间` |
| `collect_time` | ✅ | 抓取时刻 ISO 时间 |
| `source` | ✅ | 形如 `official_google` / `official_pg`（`official_` + slug） |
| `recruit_type` |  | 命中"校招/实习"用对应中文，否则留空 |
| `external_job_id` | ✅ | ATS 提供的稳定 ID；缺失则用 `md5(company|name|city|url)` |
| 其他列 |  | 留空字符串，**不要省略列名** |

## 6. 两阶段实现

### Phase 1 — 通用适配器（先通跑全部 28 家）

`GenericInhouseAdapter` 单一实现，覆盖所有 `ats_type=inhouse` 的种子。

执行流程：
1. Playwright headless 加载每条 `start_urls`
2. **滚动 + 翻页**：在主列表区域 scroll 到底，重复至 5 秒内无新链接出现，或翻到第 50 页
3. **链接抽取**：从页面所有 `<a>` 提取候选详情页 URL，命中以下任一关键词的视为岗位详情页：
   `/jobs?/`, `/job/`, `/position/`, `/career/`, `/role/`, `?jobid`, `?positionId`, `?reqId`
4. 用 `allow_domains` 白名单过滤
5. 进入每个详情页：
   - 标题：`h1` → `h2` → `[class*='title']` 顺序
   - 正文：`main` / `article` / `[class*='description']` / 全 body innerText 后兜底
   - 城市：正文或元数据命中 `上海` / `Shanghai` / `SH` 才认定
6. **过滤**：
   - `city` 必须包含上海 → 否则丢弃
   - `name + jd_raw` 至少命中一个校招/实习关键词 → 否则丢弃
7. 反爬：
   - 公司之间 sleep 3-5s；详情页之间 1-2s；带 jitter
   - UA：真实 Chrome；locale `zh-CN`；timezone `Asia/Shanghai`
   - 失败重试 3 次，指数退避（1s / 2s / 4s + 随机）

### Phase 2 — P0 专用适配器（后续单独提交）

P0 公司列表（取自 seed `priority=P0`）：

| 公司 | ATS 类型 | 备注 |
| --- | --- | --- |
| Google | inhouse | careers.google.com 有 GraphQL 端点 |
| Microsoft | inhouse | jobs.careers.microsoft.com 有 JSON 接口 |
| Amazon / AWS | inhouse | amazon.jobs/api/jobs/search |
| P&G | inhouse | 中国区入口 + 全球站点两个 start_urls 都要爬 |
| Unilever | inhouse | 上海聚合页 URL 已在 seed |
| Citadel | hedge_fund | 两个域名都要爬 |

每家提供：
- 独立的 `fetch()` — 直连 ATS 的 JSON API（如果存在），否则精确选择器
- 单元/集成测试 `tests/test_foreign_<slug>.py`：跑 1 页，断言至少 1 条 row 字段非空

## 7. CLI

```bash
python scripts/foreign_official_crawler.py [选项]

选项：
  --company NAME       只跑指定公司（按 seed 中 company 字段精确匹配）
  --priority P0|P1|P2  只跑指定优先级；可多次指定
  --industry NAME      按 industry 过滤（internet / fmcg / hedge_fund）
  --dry-run            仅打印将要爬的 URL 列表，不实际抓取
  --headed             非 headless 模式（调试用）
```

环境变量：

| 变量 | 默认 | 说明 |
| --- | --- | --- |
| `FOREIGN_OVERALL_TIMEOUT_SEC` | `28800` | 整体预算（8h） |
| `FOREIGN_COMPANY_TIMEOUT_SEC` | `900` | 单公司上限（15 min），超时跳过下一家 |
| `FOREIGN_MAX_DETAIL_PER_COMPANY` | `200` | 单公司详情页抓取上限 |

## 8. 质量门槛（验收标准）

Phase 1 通用爬完后必须满足：

- **字段填充率**（28 家合并后）：
  - `name` 非空 ≥ 99%
  - `city = 上海` 精确率 100%（不允许"上海办事处地址"等假阳性穿过）
  - `jd_raw` 非空 ≥ 90%，平均字符数 ≥ 200
  - `external_job_id` 非空 100%
- **覆盖率**：28 家中至少 22 家产出 ≥ 1 条上海岗位（剩余 6 家允许真实 0 结果，但日志必须明确标注 `0 jobs after filter`）
- **去重**：`external_job_id` 全局唯一
- **零崩溃**：单家公司失败不影响其他家继续
